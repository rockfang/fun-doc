  1. 换行符配置,CR CRLF LF转换 场景： 在文本处理中, CR, LF, CR/LF 是不同操作系统上使用的 换行符
  • Dos 和 windows 采用“回车+换行，CR/LF”表示下一行；
  • UNIX/Linux 采用“换行符，LF”表示下一行；
  • 苹果机(MAC OS 系统)则采用“回车符，CR”表示下一行

在git中add代码，常会提示：

$ git add .
warning: CRLF will be replaced by LF in ...
1
2

意思就是：我们在windows下开发的代码，换行符用的是CRLF,而git上希望存为LF Linux格式的换行符 因此就会造成别人在windows上拉取代码，拿到的是LF换行，在做diff对比代码时，带来不便。

我们可以对换行符做转换解决该问题：

# 提交时转换为LF，检出时转换为CRLF
git config --global core.autocrlf true

# 提交时转换为LF，检出时不转换
git config --global core.autocrlf input

# 提交检出均不转换
git config --global core.autocrlf false

# 本地分支操作

  1. 查看当前分支
git branch
1
  1. 创建分支
git branch new_branch
1
  1. 切换分支
git checkout 分支名
1

直接创建并切换分支：

git checkout -b v2
1
  1. 删除分支
git branch -d 分支名
1

如果分支中有修改了文件，直接删除分支可能失败。要先合并再删除

git merge new_branch
1
git branch -d new_branch
1

强行删除分支：

git branch -D new_branch
1

分支重命名：

git branch -m dev dev2
1

# 远程分支操作

查看本地及本地的远程分支状况

git branch -av
1

# 创建本地dev分支并提交到远程dev分支

  1. 创建分支：
git branch dev
1
  1. push并关联远程分支 首次关联并提交到远程dev分支：(远程会自动创建dev分支)
git push -u origin dev
1

或者这样关联：

git push --set-upstream origin dev
1

或者：(把本地v1分支关联提交到远程v1分支)

git push origin v1:v1
1

# 拉取远程的分支代码

最简单方式：(把远程v1分支拉取到本地v1分支)

git pull origin v1:v1
1
git pull origin <远程分支名>:<本地分支名>
1

方式2：

  1. 拉取远程分支信息：
git pull
1
  1. 查看分支信息:
git branch -av
1
  1. 创建并切换分支到dev:
git checkout -b dev origin/dev
1

方式3：

git checkout --track origin/dev
1

# 删除远程分支

在github上删除 命令删除： （空分支提交给远程test,实现删除远程test分支）

git push origin :test
1

或

git push origin --delete test
1
  1. clone远程指定分支代码
git clone -b dev xxx.git

# git版本库

一个项目一般常见的分支：

分支名称描述特点
master主分支，存生产代码很少变化
dev开发分支频繁改变
test测试分支，基本开发完毕给测试人员的分支常是通过dev合并提交测试用
bugfix临时修复bug的分支
  1. 查看当前项目远程的版本库 由于git是分布式，本地代码完全可以提交到2个或多个远程仓库，可通过下面命令，查看远程仓库信息：
git remote show
1

查看具体版本库的详细信息：

git remote show origin
1
  1. 克隆远程项目并改名
git clone xxx.git
1

修改拉取后，本地项目名称： myLocalName即为自己命名。

git clone xxx.git myLocalName
1

# 冲突解决

在push代码时，若远程代码有别的用户新的提交，且本地代码修改部分和远程变化部分有交叉，会引起冲突。

  1. 拉取代码并自动合并：
git pull
1

或者手动拉取合并：

git fetch
1
git merge
1
  1. 手动解决冲突: 按提示找到冲突相关的文件，进入并手动合并代码逻辑。
vi a.txt
1
  1. 重新add commit代码
git add .
1
git commit -m '解决了冲突'
1
git push
1

# git中版本穿梭

场景:想把本地代码回退到指定的某个版本 head指当前的提交点。 回退到上个版本：

git reset --hard head^
1

回退到上上个版本：

git reset --hard head^^
1

...类推

回退到上n次版本：

git reset --hard head~n
1

回退到指定版本：

git reset --hard sha1值的前几位
1

一般先通过git reflog 查看最近的操作和sha1值前几位，再 git reset --hard sha1值前几位

场景:回到过去某个版本创建分支后，基于分支写代码

  1. 查看版本提交信息获取版本sha1值
git reflog
git log
1
2
  1. 版本穿梭（游离状态），穿梭到指定版本
git checkout sha1值
1
  1. 创建分支
git branch mybranch
1
  1. 切换分支，开始代码书写
git checkout mybranch
1
  1. 回到主版本
git checkout master
1
