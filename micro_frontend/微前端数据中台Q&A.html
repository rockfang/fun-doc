# 如何正确引入图片资源

  • main.js中加入：
__webpack_public_path__ = window.__INJECTED_PUBLIC_PATH_BY_QIANKUN__;
1

目的是动态设置publicPath使得，能正确加载子应用资源。

还需要注意的是，子应用不能通过import方式引入图片，那样也会去主应用找资源，因此不建议这样使用了：🚫

import notPassUrl from "@/assets/dataQuality/notPass.png";
1

import语法会导致主应用无法加载到子应用图片，，有一个也不行

  • 官方说明中指出引入静态资源方式：使用 webpack 静态 publicPath 配置open in new window
publicPath: dev ? `//localhost:${port}` : '/',
1

效果：能解决主应用中正确加载子应用资源；副作用：子应用刷新会报favicon无法加载错误导致页面无法加载🚫

结论：

  • 配置__webpack_public_path__ = window.__INJECTED_PUBLIC_PATH_BY_QIANKUN__;
  • 子应用中不要使用import方式引入图片

颜色指标是否可自定义less变量说明
主题色是@theme-color暂时不可用，可用@primary-color对应主题色src/config/config.js文件可配置主题色
功能色是@success-color，@warning-color，@error-colorsrc/config/config.js中配置，支持hex方式，不支持rgb
文本色否@title-color，@text-color，@text-color-second文本色随主题色变化。

antd 的色系否@blue-1、 @blue-2 ... @blue-10 , @green-1、 @green-2 ... @green-10...@gold-1、 @gold-2 ... @gold-10antd-色系open in new window，此处查看完整的
