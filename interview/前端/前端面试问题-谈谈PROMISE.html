# 前端面试问题-谈谈PROMISE

Promise是ES6中处理异步的一种方案。使用范围还是比较广的。

# PROMISE的基本使用

使用方式：Promise构造函数接收一个function，这个函数中，有两个参数resolve和reject分别处理成功和失败的回调。

先看一个常规加载图片，监听回调的例子：

function loadImage(img, src) {
	img.onload = function () {
		console.log(’load success’)
	}
	img.onerror = function () {
		console.log(’load error’)
	}
	img.src = src
}

let img1 = document.getElementById(’img1’)
loadImage(img1, ’images/x.jpg’) //图片加载成功回调onload方法，加载失败，回调onerror方法。

//封装成Promise处理异步:
//用Promise包装我们的异步方法。 创建promise就开始执行我们的方法了
let p1 = new Promise(function (resolve, reject) {
	let img = document.getElementById(’img1’)
	img.src = ’images/a.jpg’
	img.onload = function () {
		resolve(img.src)
	}
	img.onerror = function () {
		reject(’load error’)
	}
})

# PROMISE的链式调用

前面的加载图片的Promise我们可以这样写：

function loadImage(img, src) {
	return new Promise(function (resolve, reject) {
		img.src = src
		img.onload = function () {
			console.log(src);
			resolve(img.src)
		}
		img.onerror = function () {
			reject(’load error’)
		}
	})
}
//即加载图片返回Promise对象。
//用链式调用实现img1加载完成再加载img2,最后加载img3:

loadImage(img1, ’images/a.jpg’)
.then(loadImage(img2, ’images/b.jpg’)
.then(loadImage(img3, ’images/c.jpg’))
)

# PROMISE.ALL的使用

实现多个异步方法同时执行，所有执行完成统一处理。

定义3个Promise

let img1 = document.getElementById(’img1’)
let img2 = document.getElementById(’img2’)
let img3 = document.getElementById(’img3’)

let p1 = loadImage(img1, ’images/a.jpg’)
let p2 = loadImage(img2, ’images/b.jpg’)
let p3 = loadImage(img3, ’images/c.jpg’)


Promise.all()处理所有异步请求：

Promise.all([p1,p2,p2]).then((res) => console.log(res));

# PROMISE.RACE用法

同Promise.all,也是多个异步方法同时执行，但Promise.race是拿到最早异步获取的结果返回。

Promise.all([p1,p2,p2]).then((res) => console.log(res));
1

将打印”/images/a.jpg”因为，上面例子中我用的图片a.jpg最小，另外两张图都很大，a.jpg加载最快。（当图片大小差不多时，可能看不出来race的效果，但返回的不再是数组。而是最早异步获取的结果）s
