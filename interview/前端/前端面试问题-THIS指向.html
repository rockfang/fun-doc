# 前端面试问题-THIS指向

this指向 当前对象。即执行这个函数的对象，谁调用这个函数this指向谁。

  1. 示例1
let btn = document.getElementById(’btn’);
btn.onclick = function() {
	console.log(this.innerHTML);
}

2
3
4

this指向 btn，即onclick这个函数的调用者。

  1. 示例2
function Person(name) {
	this.name = name;
}
let p = new Person(’cai’);

this指向当前对象：p，即调用Person构造函数者。

function Person(name) {
this.name = name;

Person() //其实是window.Person(), 方法中this指向window

  1. CALL方式调用函数，THIS指向问题 js中函数调用方式：
  • 对象.方法()
  • 方法.call(对象)
  • 方法.apply(对象) 区别是通过方法.call(对象)方式，可以改变函数的调用者，即改变this指向。 如：
Person.call(doucument)//等价于document.Person();
1

上述例子：Person构造方法中this指向document

  1. BIND使用 使用方式：let 新函数 = 函数.bind(obj) 特点：新函数的this永久指向obj。即bind函数永久改变函数this指向。 手动实现bind函数：
Function.prototype.bind = function(target) {
	let fn = this;//存bind方法的调用者，即那个函数
	return function() {
		fn.apply(target,arguments);
	}
}

解析：即创建一个bind方法，其返回一个函数，这个函数内部就是执行调用bind的哪个函数，只是通过apply改变其调用者。(此处用apply不用call是因为call需要把每个参数列出，apply可以用参数数组arguments)

  1. 箭头函数中THIS指向
  • ES6中的箭头函数完全修复了this的指向，this总是指向词法作用域，也就是指向外层调用者
  • 在箭头函数中判断this，可以当箭头函数不存在的情况下，this指向谁，那就是指向谁。

示例： 我们知道定时器中this会指向window对象

let obj = {
	id: 3,
	delayPrint:function() {
		setTimeout(function() {
			console.log(this);//将打印window对象
			console.log(this.id);//undefined 因为this并不指向obj
		},200);
	}
}
obj.delayPrint();

这是由于setTimeout()调用的代码运行在与所在函数完全分离的执行环境上。这会导致这些代码中包含的 this 关键字会指向 window (或全局)对象。 MDN说明open in new window 那如何通过this访问上述obj.id呢?箭头函数就是一种解决方式。

让SETTIMEOUT中THIS指向调用者

采用箭头函数：

let obj = {
	id: 3,
	delayPrint:function() {
		setTimeout(() => {
			console.log(this);//将打印obj对象
			console.log(this.id);//3,因为是在箭头函数中，this指向上层调用者，即obj
		},200);
	}
}
obj.delayPrint();

采用BIND方式解决

let obj2 = {
	id: 3,
	delayPrint:function() {
    	setTimeout(function () {
			console.log(this);
			console.log(this.id);
		}.bind(this),200);//bind中this即当前obj对象，因此，在调用定时器中函数时，调用者就变成了obj
	}
}

obj2.delayPrint();
